# TWYR Inline Codex 更新计划

更新日期：2026-05-10

## 产品判断

TWYR 的主体验不应继续停留在“选中文字后打开侧边栏”。新的核心体验是：

> 在 Chrome 阅读现场，选区附近直接出现一个轻量 Codex 对话框；侧边栏和独立窗口只作为深度整理、长对话、全文入库时的展开工作台。

这个调整的目标是减少阅读中断，让“疑惑、提问、回答、保存、勾连旧笔记”都发生在当前网页之上。

## 目标体验

1. 用户在任意网页选中一段文字。
2. 按 `Option+V` 打开选区旁的 Inline Codex Bubble。
3. 输入疑问后按 `Enter` 发送，`Shift+Enter` 换行，`Esc` 关闭。
4. AI 基于选区、附近上下文、页面正文、URL、标题、已有 TWYR 笔记给出回答。
5. 对话框内可继续追问，也可执行保存、查旧笔记、展开工作台。
6. `Option+S` 快速保存当前选区，不弹出大型工作台。
7. 侧边栏保留，用于长对话、全文入库、设置、复杂整理。

## 交互原则

- 默认不因选中文字自动弹出任何东西，避免干扰复制、搜索、翻译等日常操作。
- 快捷键触发后才出现对话框，且锚定在选区附近。
- 小对话框只承载即时理解和轻量追问；复杂任务一键展开到完整工作台。
- 保存不是单纯剪藏，必须记录“原文选区 + 我的提问 + AI 回答 + 来源 + 时间 + 主题标签”。
- AI 只有在问题涉及长期项目、用户主动查库、同页连续追问、或内容明显与历史主题相关时才自动检索旧笔记。

## MVP 边界

第一版先不做完整文件拖拽、多模态附件和复杂富文本编辑。输入框结构会预留附件区，但优先保证：

- 文本选区可原位提问。
- 回答能在原位展示。
- 保存和查库动作可用。
- 侧边栏可作为展开形态复用。

## 风险与处理

- **网页 CSS 污染浮层**：使用 Shadow DOM 隔离样式。
- **小窗承载不了长回答**：限制最大高度，内部滚动，并提供“展开工作台”。
- **快捷键冲突**：优先实现 `Alt+V`/`Alt+S`，如果 Chrome 不接受 `Option+V`，保留用户可在 `chrome://extensions/shortcuts` 自行绑定。
- **保存污染知识库**：默认只保存用户主动点击的内容；AI 只建议全文入库，不自动提升为 source。
- **查库过度**：检索结果必须说明引用了哪些笔记，以及为什么相关。
