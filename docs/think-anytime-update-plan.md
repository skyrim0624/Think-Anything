# Think Anytime 更新计划

更新日期：2026-05-11

## 产品目标

Think Anytime 的目标是成为一个在 Chrome 浏览途中随时可用的阅读思考层：用户看到任何文本、图片、视频画面或网页内容时，可以快速提问、追问、发散，并把真正有价值的内容沉淀进 Obsidian，之后再被 Agent 主动想起和调用。

## 本轮原则

- 先强化“随时思考 + 可再次调用”的闭环，不急着堆新入口。
- 每一步完成后必须运行检查和构建，再提交并推送。
- 对外命名统一为 `Think Anytime`；历史内部包名 `twyr` 暂不强行迁移。
- 截图素材库、完整视频多帧理解、拖拽文件等能力先作为后续路线，不进入本轮硬目标。

## 后续路线

- Harness v0：trace、feedback、Dream Proposal 和本地 eval runner，用可审计机制验证系统是否真的变聪明。
- Dock + 极速回答：把原位弹窗升级为可折叠、可拖动、默认右下角待命的阅读 Dock；默认提问走 fast 模式。
- 截图圈选后唤起 AI 讨论和保存。
- 设计偏好素材库：把海报、排版、视觉参考沉淀成可检索审美记忆。
- 视频多帧采样和时间点笔记。
- 文件和图片拖拽到 Dock。
- 更深的 Obsidian 图谱维护与 MOC 自动更新。

## Harness v0 迭代

| 步骤 | 状态 | 目标 | 交付物 |
| --- | --- | --- | --- |
| H1 | 已完成 | 记录后台判断链路 | `90-SYSTEM/harness/traces/*.jsonl`，覆盖 ask/retrieve/capture/promote/dream |
| H2 | 已完成 | 收集用户反馈 | `POST /api/feedback`，侧边栏和 Dock “有用/没用” |
| H3 | 已完成 | 生成可审计整理提案 | `POST /api/dream/propose`，`90-SYSTEM/dreams/*-proposal.md` |
| H4 | 已完成 | 建立图谱状态表 | SQLite `note_edges`、`dream_runs`、`topic_aliases` |
| H5 | 已完成 | 建立最小评测基线 | `npm run eval:harness`，输出到 `90-SYSTEM/harness/eval-runs/` |

Harness 原则：前台保持轻；后台记录完整 trace；所有深层关系先提案、后确认；每条关系必须有证据和“可能错在哪里”；被拒绝的建议保留为负反馈，不直接删除历史。
